## 📖 1. Описание задачи и постановка целей

### Проблема

Задача атрибуции авторства — одна из фундаментальных задач в области **обработки естественного языка (NLP)**. Каждый писатель обладает уникальным **идиостилем** — устойчивой совокупностью лексических, синтаксических и стилистических особенностей. Цель — автоматически распознавать эти признаки с помощью нейронной сети.

| 3 | Р. Брэдбери | Английский |

| 5 | Братья Стругацкие | Русский |

### Цели работы

1. Разработать нейросетевую модель классификации текстов по авторству
2. Достичь высокой точности (`Categorical Accuracy`) на тестовой выборке Kaggle

## 🛠️ 2. Обоснование подхода и методов

### 2.1 Предобработка данных

#### Нормализация текста (`clean_text`)

```python
text = text.lower()
text = re.sub(r'[^а-яёa-z\s]', '', text)  # только буквы
text = re.sub(r'\s+', ' ', text).strip()
```

**Мотивация:**
- Приведение к нижнему регистру — слова «Книга» и «книга» должны считаться одной лексемой
- Удаление пунктуации — убираем шум, не несущий стилистической информации
- Сохранение **обоих алфавитов** — часть авторов пишет на русском, часть на английском

#### Пословная нарезка (Word Sliding Window)

Вместо нарезки по символам применена нарезка по словам с параметрами:
- `chunk_size = 300 слов` ≈ одна страница книги — достаточно для анализа идиостиля
- `step = 150 слов` — 50% перекрытие окна = **аугментация данных без дополнительных файлов**

**Преимущество** перед посимвольной нарезкой: гарантируется целостность слов, что критично для корректной токенизации и работы эмбеддингов.

## 🏗️ 3. Архитектура модели

### Conv1D + Bidirectional LSTM (гибридная)

```
Input (300,)
  │
Embedding (VOCAB_SIZE × 128)  ← обучаемые векторные представления слов
  │           │
  │    Dropout(0.2)
  │           │
  ├──── Conv1D(128, kernel=3) ──── BatchNorm ──── MaxPool(2)  ← триграммы
  │
  └──── Conv1D(128, kernel=5) ──── BatchNorm ──── MaxPool(2)  ← пятиграммы
              │                           │
              └──────── Add ─────────────┘  ← суммирование признаков
                         │
                BiLSTM(128, bidirectional)  ← захват дальних зависимостей
                         │
                    Dropout(0.4)
                         │
                  Dense(128, relu)
                         │
                  BatchNorm + Dropout(0.3)
                         │
                  Dense(6, softmax)  ← вектор вероятностей авторов
```

### Обоснование компонентов

| Компонент | Роль |
|-----------|------|
| **Embedding** | Обучает семантическое пространство слов конкретного корпуса |
| **Conv1D (kernel=3,5)** | Параллельное извлечение n-граммных лексических паттернов |
| **Add** | Объединение признаков без увеличения размерности (vs Concatenate) |
| **Bidirectional LSTM** | Синтаксические зависимости в обоих направлениях |
| **BatchNormalization** | Стабилизация градиентов, ускорение обучения |
| **Dropout** | L2-регуляризация → снижение переобучения |

### Гиперпараметры

| Параметр | Значение | Обоснование |
|----------|----------|-------------|
| `VOCAB_SIZE` | 50 000 | Охватывает >95% уникальных слов корпуса |
| `EMBED_DIM` | 128 | Баланс выразительности и скорости обучения |
| `MAX_LEN` | 300 | P95 распределения длин (минимальные потери) |
| `BATCH_SIZE` | 64 | Оптимальный для GPU T4 |
| `LR` | 1e-3 | Adam + ReduceLROnPlateau |
| `Dropout` | 0.2 / 0.4 / 0.3 | Прогрессивная регуляризация |

### 4.2 Кривые обучения

![Training Curves](figures/training_curves.png)

Кривые демонстрируют равномерное снижение функции потерь и рост точности без признаков переобучения. Незначительный разрыв между train и val указывает на хорошую обобщающую способность модели.

### 4.3 Распределение данных

![Class Distribution](figures/class_distribution.png)

Явный дисбаланс в пользу М. Фрая (~32% данных) компенсируется взвешенной функцией потерь.

### 4.4 Матрица ошибок

![Confusion Matrix](figures/confusion_matrix.png)

**Ключевые наблюдения:**
- Модель уверенно различает **русскоязычных** и **англоязычных** авторов — ошибки почти не пересекаются между языковыми группами
- Наибольшая путаница: **О. Генри ↔ Р. Брэдбери** (оба — американские писатели, схожий базовый словарь)
- **М. Фрай** идентифицируется хуже остальных из-за широкого тематического диапазона произведений

### 4.5 Топ-слова каждого автора

![Top Words](figures/top_words_per_author.png)

Визуализация подтверждает, что модель опирается на реальные стилистические маркеры:
- **О. Генри**: акцент на диалогах, упоминание профессий
- **Булгаков**: специфическая московская лексика, театральные термины
- **Стругацкие**: научно-фантастические концепты
- **Фрай**: магические существа, специфические имена

### 4.6 XAI — Интерпретация через LIME

**LIME** (Local Interpretable Model-Agnostic Explanations) позволяет понять, какие слова в конкретном фрагменте привели к предсказанию:

- 🟢 **Зелёный** = слово указывает НА данного автора
- 🔴 **Красный** = слово указывает ПРОТИВ данного автора

**Примеры:**
- Фрагмент Булгакова: слова «воланд», «аннушка», «мастер» → высокая уверенность (0.99)
- Пограничный случай: О. Генри vs Стругацкие → ошибка на фрагменте с нейтральной лексикой

### Достигнутые результаты

✅ Разработана гибридная нейросетевая архитектура Conv1D + BiLSTM

### Направления для улучшения

| Направление | Ожидаемый эффект |
|-------------|-----------------|
| **Предобученные эмбеддинги** (FastText/RuBERT) | +2–5% точности для русских авторов |
| **Transformer-архитектура** (BERT/ruBERT) | +5–10% — более глубокий контекст |
| **Ансамблирование** нескольких моделей | снижение дисперсии ошибок |
| **Увеличение chunk_size** до 500 слов | больше контекста для сложных авторов |
| **Hard Negative Mining** | целевое улучшение для пар О.Генри/Брэдбери |
| **Data Augmentation** (синонимы, перестановки) | улучшение для миноритарных классов |
